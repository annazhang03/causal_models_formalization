
# Causality and Semantic Separation

This repository is the proof artifact for _Causality and Semantic Separation_. It formalizes
causal models in the Rocq Prover and introduces a semantic definition of _semantic separation_,
proving that it is equivalent to $d$-_separation_, the standard graphical condition used in
causal inference.

## Getting Started Guide
1. Install Docker: https://www.docker.com/products/docker-desktop/.
2. Build the Docker image using the provided `Dockerfile`. This installs dependencies and compiles the Rocq development: 
```bash
docker build -t semantic-separation .
```
3. Run the container:
```bash
docker run -it semantic-separation
```
4. Rebuild the development to verify the proofs:
```bash
make
```
5. Inspect the definition `semantically_separated` in `Semantics/SemanticSeparationDef.v`. This corresponds to
Definition 3.3 in the paper.
6. Inspect the theorem `semantic_and_d_separation_equivalent` in `Semantics/SemanticDSepEquiv.v`.
This corresponds to Theorem 3.7, the main result of the paper.
7. To verify that the proof does not rely on additional axioms, uncomment `Print Assumptions semantic_and_d_separation_equivalent.`
at the bottom of `Semantics/SemanticDSepEquiv.v`, and run `make`.
You should see the assumption of functional extensionality, as well as `contains_cycle_true_correct`, which the correctness theorem of the cycle detection function, `contains_cycle` (see Section 4 below).

## Step-by-Step Instructions

### 1. Claims Supported by the Artifact

The artifact mechanizes and verifies the main theoretical results of the paper. In particular,
the artifact contains formal Rocq definitions, and proofs where applicable, of:
- The definition of semantic separation (Definition 3.3).
- The definition of $d$-separation.
- Supporting lemmas relating graph structure and semantic behavior.
- Theorem 3.7: equivalence between semantic separation and $d$-separation.

Compiling the development checks these proofs mechanically.
The table in Section 6 maps each definition and theorem in the paper to its Rocq counterpart.

### 2. System and Build Information
- **Rocq version:** 9.1.1
- **OCaml version:** 5.4.0
- **Build system:** Makefile (auto-generated by `rocq makefile`)
- The `_CoqProject` file includes all the directories and source files necessary to build the development.

Building the Docker image will already generate the Makefile and compile the code. To do it again, one would run:
```bash

make
```

### 3. Repository Structure
- `Utils`: foundational definitions and helper lemmas for lists, logic, and type theory. 
   This directory contains general-purpose utilities that do not depend on graphs or causal concepts.
- `DAGs`: formalization of directed acyclic graphs (DAGs) in Rocq. Includes definitions of nodes, edges
   and paths, as well as key properties such as acyclicity, reachability, pathfinding, and topological
   sorting.
- `CausalDiagrams`: extensions of DAGs to causal settings. Defines notions specific to causal 
reasoning, such as _d_-separation, intermediate node classification, and interventions, building directly on the `DAGs` layer.
- `Semantics`: development of the semantic perspective on causal models. Includes the function-based
framework for causal semantics, the notion of semantic separation, and all supporting lemmas and 
theorems culminating in the fully mechanized equivalence proof between semantic separation and
_d_-separation.

### 4. Assumptions and Admitted Lemmas
- This Rocq development assumes the standard logical principle of Functional Extensionality
(imported from `Coq.Logic.FunctionalExtensionality`).
- The theorem `contains_cycle_true_correct` in `DAGs/CycleDetection.v` is currently marked as `Admitted`.
This theorem states the correctness of the `contains_cycle` function, which implements a standard cycle-detection
algorithm on graphs. The causal reasoning and semantic results in the paper rely on the existence of
such a cycle-detection procedure, and the admitted theorem concerns the correctness proof of this particular
implementation. Thus, while used in the developments in causal models and semantic reasoning,
any issue in this theorem would concern the correctness of the implementation of the cycle-detection function
rather than the formal statements or proofs built on top of it.
Work is ongoing to fully mechanize this correctness proof.
- There are a small number of other admitted theorems in `CausalDiagrams/Interventions.v` and in the `Spirtes/` directory;
these are purely auxiliary and not referenced in the semantic development or the paper.

### 5. Key Content
Our central result is a new semantic notion, semantic separation, and a proof that it is equivalent to $d$-separation.
Along the way, we provide additional definitions, lemmas, and theorems to enable this central result. Thus, to evaluate our claims, one should check that each of these intermediate results, as well as the final result, are indeed proven.

In Appendix A of the paper (duplicated in Section 6 of this document), we provide the exact mappings of definition/lemma/theorem in the paper to Rocq identifier.
In Section 6 of this document, we also provide the locations of the types defined in the paper.

Here, we provide a more in-depth guide to the most novel/important concepts in the paper and describe their appearance in the repository.

#### 5.1. Functional Semantics
In Section 3, we introduce the function-based formal semantics and use primarily mathematical notation. While we typically write assignments (e.g. unobserved-terms assignments $U$) as a function $U: \mathcal{V} \rightarrow X$, in the repository, $U$ is of type `assignments`, and we access $U(w)$ with `get_assigned_value U w`, which is defined in `CausalDiagrams/Assignments.v`. In the paper we write $f_U: \mathcal{V} \rightarrow X$ for `nodefun` $f$ with unobserved-terms assignments $U$, which in the repository is obtained with the `find_value` function and $U$ as one of the arguments, defined in `Semantics/FindValue.v`.

#### 5.2. Semantic Separation Definition
A primary contribution of this work is a novel semantic definition of $d$-separation called semantic separation. This definition appears as Definition 3.3 in the paper and a proposition called `semantically_separated` in `Semantics/SemanticSeparationDef.v` in the codebase. 

Definition 3.3 is presented with five conditions, which are analogously in the Rocq definition.

Condition 1: Lines 275-277
- $f_{U_\alpha}(u) = \alpha$ is guaranteed by 
`get_values G g Ua [] = Some Aa /\ unobs_valid_given_u G Ua Aa u a`.
- `unobs_valid_given_u` guarantees that $U_\alpha$ (`Ua`) is actually an assignment for the nodes of G, and $g_{U_\alpha}(u) = \alpha$, where `Aa` is the actual data structure that stores each of the assignments for all nodes in G.
- $f_{U_\alpha}$ properly conditions on $Z$ is given by `unobs_conditions_on_Z G g Ua AZ Z`, which is a proposition that forces each node in $Z$ to evaluate via $g$ to its assigned value in `AZ`.

Condition 2: Lines 278-281
- $f_{U_\beta}(u) = \beta$ is guaranteed by `get_values G g Ub [] = Some Ab /\ (unobs_valid_given_u G Ub Ab u b)`.
- $U_\beta$ differs from $U_\alpha$ only for $a \in \text{Anc}_Z^*(u)$ is given by the proposition `assignments_change_only_for_subset`, defined in `CausalDiagrams/Assignments.v`. It ensures that the two sets are assignments for the same nodes, and furthermore that nodes not in the given subset must evaluate to the same thing (or None) in both. 
- The `find_unblocked_ancestors` function is defined in `CausalDiagrams/UnblockedAncestors.v`, which relies on a directed pathfinding function.

Condition 3: Line 288
- The proposition `assignments_change_only_for_Z_anc_seq` sums up the entire condition recursively. It uses `assignments_change_only_for_subset` again, this time with a more complicated subset. `unblocked_ancestors_that_changed_A_to_B` extracts all the nodes that evaluate to a different value in the two assignment sets. We use `find_unblocked_ancestors_in_Z` to find unblocked ancestors of conditioned nodes which have unblocked ancestors in that set. It recurses on each triple of consecutive assignments.

Condition 4: Line 284
- `L` is the list [$U_1, ..., U_\ell$], and should have length bounded by the number of nodes in $G$.

Condition 5: 285-287
- The formalization of this condition is very similar to that of Condition 1, with the main difference being that if $\ell = 0$, then the condition applies to $U_\beta$. This is done by applying the condition on `last (Ub :: L) Ub` instead.

Finally, the value of $v$ remains unchanged is given by Line 290, `get_assigned_value Aa v = get_assigned_value Ab' v.`.

#### 5.3. Semantic Separation Equivalence Proof
The equivalence theorem between semantic separation and $d$-separation is Theorem 3.7 in the paper, and it is mechanized in `Semantics/SemanticDSepEquiv.v` as `semantic_and_d_separation_equivalent`. As explained in the paper, the proof is broken down into the two directions, each proved via contrapositive.

The precondition `generic_graph_and_type_properties_hold` is defined in `Semantics/FunctionRepresentation.v`, and it adds that $G$ must be acyclic, there are at least two distinct members of type `X`, and the graph itself is well-formed in Rocq, e.g. all nodes in edges are also in the set of nodes, and each node/edge appears at most once. We require similarly that nodes in `Z` appear at most once.

It should otherwise be clear that `semantic_and_d_separation_equivalent` corresponds to Theorem 3.7. Uncommenting the line `Print Assumptions semantic_and_d_separation_equivalent.` will show that the only assumptions are as described above in Section 4.
